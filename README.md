#  アウトライン  
## ・作成目的  
	⇒何を伝えたいのか？  
		⇒誰に対する何のためのガイドライン？  
			⇒SaaS型CI/CDツールとCloudを組み合わせた開発環境構築の指針を提示することを目的とする  
  
## ・作成背景  
	⇒なぜ、SaaS型CI/CDツールを利用する？  
		⇒Jenkinsだと、属人化しがちでメンテナンス性・コストがかかる  
			⇒SaaS型CI/CDツールなら改善できそう  
		
	⇒なぜ、AlibabaCloudを利用する。  
		⇒Publicクラウドの需要はますます高まっているが、AWS一強の状態は変わりつつある  
			⇒マルチクラウド対応が求められる。  
				⇒使ってみよう  
				
## ・環境の全体像  
	⇒クライアントPC（作業者）  
		⇒チャットツールに接続してCI/CD結果の確認を行う  
		⇒クラウド上のNexus・SonarQubeに接続してライブラリのバージョン管理、CI/CDのテスト結果の確認を行う  
		⇒Webアプリを利用する  
		⇒GitHubに改修したソースを格納する  
		  
	⇒チャットツール（Slack）  
		⇒パイプライン実行結果を受け取る  
		
	⇒バージョン管理ツール  
		⇒ソースのバージョン管理を行う  
		  
	⇒CI/CDツール  
		⇒バージョン管理ツール上の操作を契機として、対象リポジトリのソースに対してビルド確認・Junitテスト/静的検証、デプロイを行う  
		⇒テスト結果のSonarQube連携、ライブラリのNexus連携を行う  
		  
	⇒Cloud  
		⇒各種アプリケーションの運用を行う。  
		  
## ・Cloud環境の整備  
	⇒概要  
		⇒Cloud環境には、Webアプリケーション実行用の環境とSonarQube・Nexus実行用の環境を準備する。  
	⇒ネットワーク構成  
		⇒各サーバは、ロードバランサを使ってパスルーティングを行う  
	⇒サーバ構成  
		⇒Webアプリケーション実行用のサーバ  
			⇒どうやってWebアプリケーションを実行しているか  
				⇒起動Shellの作成  
				⇒Shellのサービス登録  
				⇒systemctlによるサービス実行  
		⇒SonarQube・Nexus用サーバ  
			⇒各アプリケーションの実行方法  
			⇒各アプリの設定  

## ・CI/CDツールの設定  
	⇒概要  
		⇒対象リポジトリのルートディレクトリ配下に設定ファイルを配置することで、GitHubへの特定の動作を契機としてCI/CDが行われる  
		⇒特にCircleCIについて記述する。

		⇒どこに配置する？  
		⇒どう記述する？
